1 Who We Are

Seven ("we", "us", "our") develops StatusWatch for Jira, an Atlassian Marketplace app built on the Atlassian Forge platform. This policy covers how we handle personal data when the app is installed on a Jira Cloud instance.

It does not apply to Atlassian's own services — see the Atlassian Privacy Policy for those.

2 Data We Collect

We store only what is necessary to deliver the app's features.

Data Purpose Storage
Jira Account ID Audit log authorship; access control allowlist Forge KV Storage
Display Name Human-readable audit log entries Forge KV Storage
Jira Group ID Group-based access control Forge KV Storage
Avatar URL Shown during admin user search Not stored — fetched on demand only
Statuspage API Key Authenticating to private Statuspage instances Forge Secret Storage (encrypted)
✅ We do not collect email addresses, IP addresses, or usage analytics. Incident data fetched from Statuspage contains no personal information.

3 Legal Basis for Processing

  • Legitimate interests — Audit log records serve the administrator's interest in tracking configuration changes.
  • Contract performance — Account IDs in access settings are required to deliver the access control feature.
  • Implicit consent — Administrators acknowledge data storage when adding users or groups to the access list.

4 How We Use Your Data

  • Audit logging — Recording which administrator changed a connection, access setting, or banner, with their name and timestamp.
  • Access control — Determining which Jira users or groups can view the StatusWatch status page.

We do not use personal data for advertising, profiling, or analytics.

5 Data Storage & Security

All data is stored in Atlassian Forge KV Storage and Forge Secret Storage — Atlassian-managed services within the Atlassian Cloud.

🔒 No personal data leaves the Atlassian platform. We do not operate our own databases or cloud storage.
  • API keys are encrypted at rest via Forge Secret Storage
  • All resolver functions verify the caller's Jira permissions before processing
  • External domains are explicitly allowlisted in the app manifest
  • Incoming webhooks are verified with HMAC-SHA256 signature checking

6 Data Retention

Data Retention Deletion
Audit Log Rolling — oldest entries removed automatically at 500-entry limit Automatic
Access Settings Until removed by a site administrator Manual, via app settings
Statuspage API Key Until the connection is deleted Manual, via app settings
Incident Cache 90 days for resolved incidents Automatic

Uninstalling the app removes all associated data from Forge Storage per Atlassian's data lifecycle policy.

7 Third-Party Services

The app fetches incident data from Statuspage.io (operated by Atlassian). This is read-only and involves no personal data — only the configured page domain is transmitted.

No personal data is shared with any advertising network, analytics platform, or third party.
