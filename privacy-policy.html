2 Scope of This Policy

This Privacy Policy applies to StatusWatch for Jira and describes how we handle personal data collected through the application when installed on a Jira Cloud instance via the Atlassian Marketplace.

This policy does not apply to Atlassian's own products and services. For Atlassian's privacy practices, please refer to the Atlassian Privacy Policy.

3 Data We Collect

StatusWatch for Jira collects only the minimum data necessary to provide its features. The table below describes each category of personal data stored by the application.

Data Description Purpose Stored?
Jira Account ID Unique identifier assigned by Atlassian to each Jira user Audit log authorship; access control allowlist Yes — Forge KV Storage
Display Name The name shown in Jira for each user Human-readable audit log entries Yes — Forge KV Storage
Jira Group ID Identifier of a Jira user group Group-based access control Yes — Forge KV Storage
Avatar URL Profile picture URL from Atlassian Displayed in the admin UI during user search No — fetched at request time only, never persisted
Statuspage API Key API credential for private Statuspage instances Authenticating to private Statuspage pages Yes — Forge Secret Storage (encrypted)
✅ We do not collect names, email addresses, IP addresses, device information, usage analytics, or any data beyond what is listed above. Incident data fetched from Statuspage is operational data — it contains no personal information.

4 Legal Basis for Processing

We process personal data on the following legal bases under GDPR and applicable privacy law:

  • Legitimate interests — Storing Account IDs and display names in the audit log serves the legitimate interest of the Jira site administrator in maintaining an auditable record of configuration changes.
  • Contract performance — Storing Account IDs in access settings is necessary to deliver the access control feature that the Jira administrator has configured as part of using the application.
  • Consent (implicit) — By adding a user or group to the access list, the Jira administrator acknowledges that those identifiers will be stored.

5 How We Use Your Data

Personal data is used solely for the following purposes:

  • Audit logging — Recording which Jira administrator added or removed a Statuspage connection, changed access settings, or triggered a banner, with their name and timestamp.
  • Access control — Determining which Jira users or groups are permitted to view the StatusWatch status page within Jira.

We do not use personal data for advertising, profiling, analytics, or any purpose beyond the functionality described above.

6 Data Storage & Security

All data stored by StatusWatch for Jira resides exclusively in Atlassian Forge KV Storage and Forge Secret Storage. These are Atlassian-managed infrastructure services hosted within the Atlassian Cloud.

🔒 No personal data leaves the Atlassian platform. We do not operate our own databases, servers, or cloud storage. Data does not pass through any third-party infrastructure owned or operated by us.

Security measures in place:

  • API keys are stored in Forge Secret Storage, which encrypts secrets at rest
  • All resolver functions validate the calling user's Jira permissions before processing requests
  • External fetch URLs are explicitly allowlisted in the application manifest — the app cannot communicate with any unlisted domain
  • Webhook signatures are verified using HMAC-SHA256 before processing incoming payloads

7 Data Retention

Data Retention Period Deletion Method
Audit Log (Account ID + Display Name) Rolling window — oldest entries removed automatically when the 500-entry limit is reached Automatic; no manual action required
Access Settings (Account ID, Group ID) Until explicitly removed by a Jira site administrator Admin removes the user or group from the access list in the app settings
Statuspage API Key Until the connection is deleted by an administrator Admin removes the connection in the app settings
Incident Cache 90 days for resolved incidents; active incidents cleared on resolution Automatic

When the application is uninstalled from a Jira site, all data stored in Forge KV Storage associated with that installation is deleted in accordance with Atlassian's Forge data lifecycle policies.

8 Third-Party Services

StatusWatch for Jira communicates with Statuspage.io (operated by Atlassian) to fetch incident data for the services you configure. This communication is read-only and involves no personal data — only the domain or page identifier of the Statuspage instance is transmitted.

The application does not integrate with any advertising networks, analytics platforms, or data brokers. No personal data is shared with any third party.

ℹ️ All external domains the app is permitted to contact are declared in the application's manifest.yml file and reviewed by Atlassian as part of the app listing process.
